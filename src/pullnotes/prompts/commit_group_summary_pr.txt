# Prompt: Resumir um grupo de commits em bullets para descricao de Pull Request.

You need to summarize a compilation of "{{change_type_label}}" type changes for a Pull Request description.

Create a list of bullet points that summarizes all the changes made by the commits, providing TECHNICAL DETAILS.

## Guidelines for PR summaries:
- Use the Change Summary section to understand what was modified
- Keywords indicate the main concepts added or removed
- Artifacts show structural changes (APIs, events, services)
- Include technical implementation details (classes, methods, modules affected)
- Mention specific technologies, frameworks, or libraries used
- Reference architectural changes or patterns applied

Keep the order of the commits, as they are sorted by level of importance (most important first).

The final text should be in {{language_hint}}

## Anti-hallucination rules:
- Use ONLY information explicitly present in the commit list below.
- Do NOT invent, assume, or infer technical details not stated in the commits (e.g., class names, methods, libraries, patterns, or architectural decisions not mentioned).
- Do NOT add context from your own knowledge about frameworks, tools, or common practices.
- If a commit lacks technical details, summarize only what is explicitly stated without fabricating implementation specifics.
- Prefer to omit information rather than risk stating something not supported by the data.

Return ONLY the bullet points in the following format:
- Brief technical summary of the change (1-2 sentences with implementation details)

Do NOT include commit SHA or any other metadata in the output.
