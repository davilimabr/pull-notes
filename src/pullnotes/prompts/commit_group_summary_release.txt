# Prompt: Resumir um grupo de commits em bullets para Release Notes.

You need to summarize a compilation of "{{change_type_label}}" type changes for Release Notes.

Create a list of bullet points that summarizes all the changes made by the commits, focusing on USER-VISIBLE FUNCTIONALITY.

## Guidelines for Release Notes summaries:
- Focus on what functionality changed, not how it was implemented
- Use language that non-technical users can understand
- Emphasize the value and impact of the changes
- Avoid technical jargon, implementation details, class names, or method names
- Describe the feature or fix from a user's perspective
- Mention the benefit or problem solved when relevant

Keep the order of the commits, as they are sorted by level of importance (most important first).

The final text should be in {{language_hint}}

## Anti-hallucination rules:
- Use ONLY information explicitly present in the commit list below.
- Do NOT invent, assume, or infer user-facing impacts, benefits, or behaviors not stated in the commits.
- Do NOT add context from your own knowledge about how features typically work or what users might expect.
- If a commit's user-visible impact is unclear, describe only what is explicitly stated without guessing user experience.
- Prefer to omit information rather than risk stating something not supported by the data.

Return ONLY the bullet points in the following format:
- Brief functional summary of the change (1-2 sentences focusing on what users will experience)

Do NOT include commit SHA or any other metadata in the output.
