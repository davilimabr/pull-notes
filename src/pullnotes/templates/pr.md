# Titulo do Pull Request

## Descricao

Descreva de forma clara e objetiva o que foi alterado e por que. Inclua o contexto necessario para que o revisor entenda a motivacao da mudanca.

- **Problema**: Qual problema ou necessidade originou esta alteracao?
- **Solucao**: Qual abordagem foi adotada para resolver o problema?

## Tipo de Alteracao

Marque o tipo de alteracao que este pull request introduz:

- [ ] Correcao de bug
- [ ] Nova funcionalidade
- [ ] Alteracao que quebra compatibilidade (breaking change)
- [ ] Refatoracao (sem alteracao de comportamento)
- [ ] Atualizacao de documentacao
- [ ] Melhoria de performance
- [ ] Atualizacao de dependencias
- [ ] Configuracao de infraestrutura ou CI/CD

## Alteracoes Realizadas

Liste as principais alteracoes realizadas, agrupadas por escopo quando aplicavel.

## Riscos e Impactos

Descreva riscos conhecidos, efeitos colaterais ou areas do sistema que podem ser afetadas por esta alteracao.

- Impacto em APIs existentes: Sim / Nao
- Impacto em banco de dados: Sim / Nao
- Requer migracoes: Sim / Nao
- Requer atualizacao de variaveis de ambiente: Sim / Nao

## Plano de Testes

Descreva como as alteracoes foram validadas.

- [ ] Testes unitarios adicionados ou atualizados
- [ ] Testes de integracao adicionados ou atualizados
- [ ] Testes manuais realizados (descreva os cenarios abaixo)


## Checklist

- [ ] O codigo segue os padroes e convencoes do projeto
- [ ] Nao ha credenciais, segredos ou dados sensiveis no codigo
- [ ] As alteracoes foram testadas localmente
- [ ] A documentacao foi atualizada (se aplicavel)
- [ ] As dependencias foram atualizadas no arquivo de configuracao (se aplicavel)
